• SQL Insurance Claims Analysis Project

- Project Overview

This project focuses on analyzing insurance claims data using SQL to simulate real-world data analysis workflows. The objective is to explore customer behavior, claim patterns, and risk indicators through structured relational database design and querying.

The project demonstrates how raw data can be transformed into meaningful insights using SQL techniques commonly used in entry-level data analyst roles.

---

- Objectives

- Understand insurance claims and customer data relationships
- Practice SQL-based data cleaning and transformation
- Build relational queries using multiple tables
- Generate business insights through aggregations and analysis
- Simulate a basic ETL workflow using SQL

---

- Dataset Description

The project uses two main datasets:

- **Customers Table**: Contains customer demographic and lifestyle information
- **Claims Table**: Contains insurance claim records linked to customers

These datasets are connected using relational keys (customer_id), allowing for multi-table analysis.

---

- Key SQL Work Completed

- Creation of relational database tables
- Data cleaning and standardization
- Use of foreign keys to establish relationships
- Implementation of JOIN operations between tables
- Aggregation of data using GROUP BY
- Creation of derived fields for analytical purposes
- Development of summary tables for reporting

---

- Analytical Focus Areas

- Total claim amounts by customer segments
- Claim frequency and severity analysis
- Risk-based classification of claims
- Time-based analysis of claims (year and month trends)
- Identification of high-value customers and policies

---

- Key Learnings

- Understanding relational database design principles
- Applying SQL for real-world data analysis tasks
- Working with structured datasets and multi-table relationships
- Building analytical thinking through business-oriented queries
- Developing foundational ETL-style workflows using SQL

---

- Conclusion

This project represents a complete SQL-based data analysis workflow, starting from raw data structure to final analytical outputs.

It demonstrates practical skills in SQL, data modeling, and business-oriented analysis, which are essential for junior data analyst and reporting roles.
